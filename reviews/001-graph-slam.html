SLAM GRAPH THEORY OPTIMIZATION

A Tutorial on Graph-Based SLAM

Giorgio Grisetti et al. • IEEE Intelligent Transportation Systems Magazine (2010)

EXECUTIVE SUMMARY

// THE PROBLEM

Mobile robots need accurate maps to navigate without external reference (GPS). However, sensor measurements are inherently noisy. This creates the Simultaneous Localization and Mapping (SLAM) problem, where errors accumulate over time, leading to inconsistent maps.

// THE SOLUTION

The paper proposes a "Graph-Based" formulation. Robot poses are treated as Nodes (pins) and measurements as Edges (springs). The problem is solved by minimizing the "tension" in these springs (Least Squares Error Minimization) using Gauss-Newton or Levenberg-Marquardt algorithms.

// THE RESULT

A robust "Back-end" solution that exploits the sparse structure of the SLAM matrix. It demonstrates efficient map correction (e.g., Intel Lab, MIT Killian Court datasets) even with thousands of nodes, running in real-time on standard laptops.

STRENGTHS +

  • Didactic Clarity: Converts complex probabilistic math into intuitive "spring-mass" physical models.
  • Efficiency: Clearly explains how to use the sparse structure of the Information Matrix ($H$) to speed up computation.
  • 3D Robustness: Addresses singularities in 3D rotations by optimizing on a manifold instead of Euclidean space.

WEAKNESSES -

  • Back-end Only: Assumes the "Front-end" (Data Association) is perfect. Does not solve perceptual aliasing.
  • Static World Assumption: The math assumes the environment is static. Moving objects are treated merely as noise/outliers.
